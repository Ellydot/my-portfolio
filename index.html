About

문제를 이해한 뒤,
구조부터 정리합니다

저는 시각적 완성도만이 아니라, 사용자가 어떤 맥락에서 화면을 보고 무엇을 빠르게 판단해야 하는지부터 고민합니다.

관리자 페이지나 태블릿 UI처럼 목적이 분명한 화면에서 복잡한 기능도 한눈에 이해되도록 정돈된 인터페이스로 구조화하는 데 집중합니다.

최근에는 AI를 아이데이션과 문서화, 구조 설계까지 확장하는 실무 파트너로 활용하며 더 빠르고 넓은 시야로 문제를 해결합니다.

정보 구조 설계 태블릿 UI 관리자 UX Figma AI 협업 비주얼 설계
01구조 우선디자인 전에 정보 계층과 흐름을 먼저 정리합니다
02맥락 파악실제 사용 환경과 사용자 목표를 깊이 이해합니다
03전달 가능성개발자가 바로 구현할 수 있는 명확한 스펙을 제공합니다
04지속적 개선피드백과 데이터를 바탕으로 반복적으로 개선합니다
